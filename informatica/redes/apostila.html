O endereço IP é um código numérico que identifica cada dispositivo em uma rede — como um "CEP" digital. Sem ele, os dados não saberiam para onde ir.

IPv4 — 32 bits

Escrito em 4 grupos decimais separados por ponto. Cada grupo vai de 0 a 255.

Exemplo: 192.168.1.10
Formato: XXX.XXX.XXX.XXX → 4 octetos × 8 bits = 32 bits
Total de endereços: ~4,3 bilhões
IPv6 — 128 bits

Escrito em 8 grupos hexadecimais separados por dois pontos. Criado para suprir o esgotamento do IPv4.

Exemplo: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
Formato: XXXX:XXXX:XXXX:XXXX:XXXX:XXXX:XXXX:XXXX
Total de endereços: ~340 undecilhões (3,4 × 10³⁸)

LAN (Local Area Network) é uma rede que conecta dispositivos em uma área geográfica pequena — sala, andar, prédio ou campus. Alta velocidade, baixa latência, gerenciada pela própria organização.

TipoNome completoAlcanceExemplo
LANLocal Area NetworkSala / prédioLaboratório de informática
MANMetropolitan Area NetworkCidadeRede de uma universidade
WANWide Area NetworkPaís / mundoInternet

Dispositivos comuns em uma LAN: switch (interliga dispositivos), roteador (conecta à internet), access point (Wi-Fi), computadores, impressoras.

💡 Em uma LAN, dispositivos com IPs na mesma faixa se comunicam diretamente, sem precisar do gateway.

VPN (Virtual Private Network) cria um túnel criptografado através da internet, permitindo que dispositivos remotos se conectem a uma rede privada com segurança — como se estivessem presentes fisicamente.

Como funciona
1. Dispositivo do usuário criptografa o tráfego
2. Envia pelo túnel VPN pela internet
3. Servidor VPN da empresa descriptografa
4. Acessa os recursos internos normalmente
Tipos de VPN
  • Acesso remoto: funcionário em casa acessa a rede da empresa
  • Site-to-site: interliga filiais de uma empresa como se fossem uma única LAN

DHCP Reservation

Associa o MAC address (endereço físico) de um dispositivo a um IP específico. Resultado: o dispositivo sempre recebe o mesmo IP automaticamente — sem configuração manual.

Sem DHCP: APIPA
Se o servidor DHCP não responder, o Windows atribui automaticamente um endereço APIPA: 169.254.x.x
Com APIPA: sem gateway, sem DNS, sem acesso à rede ou internet

DNS (Domain Name System) traduz nomes de domínio em endereços IP — funciona como a "lista telefônica" da internet.

Você digita: www.google.com
DNS resolve: 142.250.218.14
Navegador acessa: 142.250.218.14
Servidores DNS públicos
8.8.8.8 e 8.8.4.4 → Google
1.1.1.1 → Cloudflare
DNS Forwarder

Servidor DNS interno que, ao não conhecer um domínio externo, repassa a consulta para um servidor DNS externo e retorna a resposta ao usuário. Combina resolução local (servidores internos) com acesso externo (internet).

Bloqueio de sites pelo DNS

O servidor DNS pode ser configurado para não resolver determinados domínios — retornando erro quando alguém tenta acessar o site pelo nome. Funciona mesmo que o IP do site mude frequentemente.

⚠ Sem DNS: o site só abre se digitar o IP diretamente. Navegar por nomes fica impossível.

O gateway padrão é o dispositivo (geralmente o roteador) que serve como "portão de saída" da rede local para outras redes — incluindo a internet.

🏢 Analogia: o gateway é a portaria de um condomínio. Pacotes internos (mesma rede) não precisam passar pela portaria. Pacotes externos passam obrigatoriamente.

TCP
Transmission Control Protocol
  • Orientado a conexão (handshake)
  • Garante entrega de todos os pacotes
  • Garante ordem correta
  • Detecta e corrige erros
  • Mais lento (overhead de confirmações)
Uso: HTTP/HTTPS, e-mail, banco online, FTP, download de arquivos
UDP
User Datagram Protocol
  • Sem conexão prévia
  • Sem garantia de entrega
  • Sem garantia de ordem
  • Sem verificação de erros
  • Muito rápido (sem overhead)
Uso: streaming de vídeo, VoIP, jogos online, consultas DNS
Portas e Sockets

O TCP identifica cada conexão por um socket: combinação de IP de origem + porta de origem + IP de destino + porta de destino. As portas de origem são geradas automaticamente pelo SO (ephemeral ports, geralmente > 1024), permitindo que dois clientes acessem o mesmo servidor simultaneamente sem confusão.

💡 Regra simples: dados onde a perda é aceitável → UDP. Dados onde a integridade é crítica → TCP.

NAT (Network Address Translation) permite que múltiplos dispositivos com IPs privados compartilhem um único IP público para acessar a internet.

Como funciona
1. PC interno (192.168.1.10:52000) envia pacote para google.com
2. Roteador substitui pelo IP público (200.10.1.1:45001)
3. Google responde para 200.10.1.1:45001
4. Roteador consulta tabela → encaminha para 192.168.1.10:52000

O roteador usa portas diferentes para distinguir cada dispositivo.
🏢 Analogia: edifício onde todos os funcionários usam o mesmo ramal externo, mas cada um tem um ramal interno diferente. Para quem liga de fora, o número é sempre o mesmo.
NAT e IP Público/Privado
✦ Sem NAT: cada dispositivo precisaria de um IP público próprio
✦ Com NAT: 40 computadores com IPs privados → 1 único IP público
✦ Sites externos veem sempre o IP público do roteador

A máscara de sub-rede define qual parte do endereço IP identifica a rede e qual parte identifica o host (dispositivo).

Notação CIDR

O número após a barra (/) indica quantos bits estão "ligados" na máscara (identificando a rede).

/8= 255.0.0.0 → 8 bits de rede · 24 bits de host → ~16,7 milhões de dispositivos
/16= 255.255.0.0 → 16 bits de rede · 16 bits de host → 65.534 dispositivos
/24= 255.255.255.0 → 24 bits de rede · 8 bits de host → 254 dispositivos
Identificar se dois IPs estão na mesma rede
Com máscara /24, os 3 primeiros octetos devem ser iguais:

192.168.1.10 e 192.168.1.50 → mesma rede ✅ se comunicam diretamente
192.168.1.10 e 192.168.2.10 → redes diferentes ❌ precisam do gateway
/24 em binário:
11111111.11111111.11111111.00000000
   8 bits   +   8 bits   +   8 bits   +   0 bits = 24
