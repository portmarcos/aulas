O que é Linux e Linux Mint
Kernel · Distribuições · Edições Mint
  • Linux é um kernel de código aberto criado por Linus Torvalds em 1991
  • Linux Mint é uma distribuição baseada no Ubuntu/Debian, fácil de usar
  • Edições: Cinnamon (padrão), MATE (leve) e Xfce (muito leve)

Kernel: núcleo do sistema operacional. O Linux é o kernel; a "distribuição" é o sistema completo (kernel + programas + interface).

Linux Mint: lançado em 2006, baseado no Ubuntu (que é baseado no Debian). Foco em facilidade de uso para quem vem do Windows.

Edições: Cinnamon → interface moderna (padrão, mais recursos); MATE → clássico e rápido; Xfce → ideal para hardware muito antigo.

Open source: o código-fonte é público, pode ser estudado, modificado e redistribuído gratuitamente. Licença GPL.

Interface Gráfica — Desktop
Cinnamon · Painel · Menu · Gerenciador de Janelas
  • Cinnamon: painel na base, menu iniciar, bandeja do sistema
  • Área de trabalho com ícones, janelas e múltiplos espaços de trabalho
  • Centro de Controle: configura tema, rede, som, usuários, impressoras

Painel (taskbar): barra inferior com menu iniciar (🌿), janelas abertas, relógio e bandeja do sistema. Similar à barra de tarefas do Windows.

Menu Mint: organizado por categorias (Internet, Escritório, Gráficos…). Possui campo de busca rápida para abrir programas.

Espaços de trabalho: o Cinnamon permite múltiplas áreas de trabalho virtuais — como ter várias telas em uma.

Centro de Controle: equivalente ao Painel de Controle do Windows. Permite configurar aparência, rede, usuários, hardware e mais.

Sistema de Arquivos
Estrutura de diretórios · /home · /etc · /usr
  • No Linux, tudo parte de / (raiz) — não há letras de drive (C:, D:)
  • Diretórios principais: /home (usuários), /etc (configurações), /usr (programas)
  • Caminhos: absoluto (/home/marcos) ou relativo (../Documentos)

Estrutura hierárquica: tudo começa em / (raiz). Ao contrário do Windows, não existem C:, D:.

Diretórios essenciais:
/home/usuario → arquivos pessoais do usuário (equivale a C:\Users\)
/etc → arquivos de configuração do sistema
/usr → programas instalados
/var → logs, banco de dados, e-mail
/tmp → temporários (apagados no reinício)
/root → home do superusuário root
/dev → dispositivos de hardware como arquivos

~ (til) representa a pasta home do usuário atual: ~/Documentos = /home/marcos/Documentos.

Terminal — Comandos Essenciais
ls · cd · mkdir · rm · cp · mv · cat · man
  • ls: lista arquivos e diretórios; ls -la: mostra detalhes e ocultos
  • cd: navega entre diretórios; pwd: mostra o diretório atual
  • mkdir/rm/cp/mv: criar, remover, copiar e mover arquivos

Navegação: pwd (onde estou?), cd /caminho (entrar numa pasta), cd .. (voltar um nível), cd ~ (ir para home).

Listagem: ls (arquivos), ls -l (detalhes), ls -a (inclui ocultos), ls -la (ambos).

Manipulação: mkdir pasta (criar diretório), rm arquivo (apagar), rm -rf pasta (apagar recursivo — cuidado!), cp origem destino (copiar), mv origem destino (mover/renomear).

Leitura: cat arquivo (mostrar conteúdo), less arquivo (paginar), nano arquivo (editar).

Ajuda: man comando (manual completo), comando --help (resumo rápido), whatis comando (descrição em uma linha).

5
Gerenciamento de Pacotes
apt · apt-get · Software Manager · dpkg
  • APT (Advanced Package Tool): instala/remove/atualiza programas via terminal
  • sudo apt update: atualiza a lista de pacotes disponíveis
  • sudo apt install nome: instala; sudo apt remove nome: remove

Fluxo de instalação: o APT busca no repositório → baixa o pacote → instala automaticamente com todas as dependências.

Comandos principais:
sudo apt update → atualiza a lista de programas disponíveis
sudo apt upgrade → atualiza todos os pacotes instalados
sudo apt install nome → instala um programa
sudo apt remove nome → remove (mantém configurações)
sudo apt purge nome → remove completamente (inclusive configurações)
sudo apt autoremove → remove dependências não utilizadas

Interface gráfica: o Software Manager (Gerenciador de Programas) do Mint oferece a mesma funcionalidade com interface visual — equivale à Microsoft Store.

Repositórios: servidores onde ficam os pacotes. Configurados em /etc/apt/sources.list. O Mint usa os repositórios do Ubuntu + repositórios próprios.

Usuários, sudo e Permissões
chmod · chown · sudo · adduser · grupos
  • sudo: executa um comando como superusuário temporariamente
  • chmod: altera permissões — rwx (leitura, escrita, execução)
  • Notação octal: 7=rwx, 6=rw-, 5=r-x, 4=r--, 0=---

Leitura de permissões: -rwxr-xr-x = tipo + dono (rwx) + grupo (r-x) + outros (r-x).
r=leitura(4), w=escrita(2), x=execução(1). Soma: rwx=7, rw-=6, r-x=5, r--=4.

chmod: chmod 755 script.sh → dono pode tudo (7), grupo pode ler/executar (5), outros idem (5).
chmod +x script.sh → adiciona permissão de execução para todos.

sudo vs root: sudo permite executar um comando como root temporariamente, com registro em log. É mais seguro do que logar como root diretamente.

Gerenciamento de usuários:
sudo adduser aluno → cria usuário
sudo passwd aluno → define senha
sudo usermod -aG sudo aluno → adiciona ao grupo sudoers
whoami → mostra o usuário atual

Gerenciamento de Processos
ps · top · htop · kill · killall
  • ps aux: lista todos os processos em execução com PID e uso de CPU
  • top / htop: monitor interativo em tempo real (htop é mais visual)
  • kill PID: encerra processo pelo número; killall nome: pelo nome

ps aux: USER, PID, %CPU, %MEM, COMMAND. Use ps aux | grep firefox para filtrar.

top: atualiza a cada 3 segundos. Pressione q para sair, k para matar um processo, M para ordenar por memória.

htop: versão visual e colorida do top. Instale com sudo apt install htop. Permite clicar nos processos com o mouse.

Sinais de término:
kill PID → envia SIGTERM (pedido educado de encerramento)
kill -9 PID → envia SIGKILL (forçado, sem possibilidade de ignorar)
killall firefox → mata todos os processos com esse nome

Rede e Conectividade
ip addr · ping · ifconfig · DNS · Firewall
  • ip addr show: exibe interfaces e endereços IP (substitui ifconfig)
  • ping hostname: testa conectividade com um host na rede
  • Network Manager: interface gráfica para configurar Wi-Fi e cabeado

Verificar rede:
ip addr show (ou ip a) → lista interfaces com seus IPs
ifconfig → versão mais antiga (instalar: sudo apt install net-tools)
ip route → mostra o gateway padrão

Diagnóstico:
ping google.com → testa conectividade (Ctrl+C para parar)
traceroute google.com → mostra o caminho dos pacotes
nslookup google.com → resolve DNS
wget URL ou curl URL → baixa arquivo/conteúdo

Network Manager: ferramenta gráfica do Mint para configurar conexões Wi-Fi, VPN, DNS e Ethernet sem precisar do terminal.

Linux Mint vs Windows 10
Custo · Desempenho · Segurança · Compatibilidade
  • Linux Mint: gratuito, open source, seguro, leve — ideal para PCs antigos
  • Windows 10: licença paga, mais softwares comerciais, interface familiar
  • Linux domina em servidores; Windows domina em desktops corporativos

Custo: Linux Mint é 100% gratuito e continuará sendo. Windows 10 requer licença (Home ~R$500, Pro ~R$900).

Hardware: Mint com Xfce roda bem em PCs com 1 GB RAM e processador de 1 GHz. Windows 10 requer oficialmente 2 GB RAM e mais recursos.

Segurança: vírus para Linux são raros (arquitetura de permissões dificulta infecção). Windows é alvo frequente de malware por ter maior base de usuários.

Software: Mint tem LibreOffice, GIMP, VLC, Firefox e mais de 50.000 programas no repositório. Jogos e alguns softwares profissionais ainda têm melhor suporte no Windows.

Uso no mercado: Linux roda em ~96% dos servidores do mundo, em supercomputadores, Android (kernel Linux) e nuvem. No desktop, Windows ainda domina (~75%), mas Linux cresce.
